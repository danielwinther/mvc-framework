81 ### Fixed

83 * Merged [#1885](https://github.com/sebastianbergmann/phpunit/issues/1885): Fixed handling of PHP configuration settings for process isolation
84 * An outdated version of DbUnit was bundled in the PHAR distribution of PHPUnit

86 ## [5.0.0] - 2015-10-02
87 
88 ### Added
89 
90 * Implemented [#1604](https://github.com/sebastianbergmann/phpunit/issues/1604): A `@small` test should be marked as risky when it executes code that performs I/O operations
91 * Implemented [#1656](https://github.com/sebastianbergmann/phpunit/issues/1656): Allow sorting test failures in reverse
92 * Merged [#1753](https://github.com/sebastianbergmann/phpunit/issues/1753): Added the `assertFinite()`, `assertInfinite()` and `assertNan()` assertions
93 * Merged [#1876](https://github.com/sebastianbergmann/phpunit/issues/1876): Added the `--atleast-version` commandline option
94 * Implemented [#1780](https://github.com/sebastianbergmann/phpunit/issues/1780): Support for deep-cloning of results passed between tests using `@depends`
95 * Implemented [#1821](https://github.com/sebastianbergmann/phpunit/issues/1821): Expectations on mock objects passed via `@depends` are now also evaluated for the depending test
96 * Added `--whitelist` commandline option to configure a whitelist for code coverage analysis
97 * Added convenience wrapper `getMockWithoutInvokingTheOriginalConstructor()` to create a test double without invoking the original class' constructor
98 * Added TeamCity test result logger for more seamless integration of PHPUnit with PhpStorm
99 
100 ### Changed
101 
102 * Merged [#1781](https://github.com/sebastianbergmann/phpunit/issues/1781): Empty string is not treated as a valid JSON string anymore
103 * Merged [#1822](https://github.com/sebastianbergmann/phpunit/issues/1822): Always output progress totals on last line
104 * It is now mandatory to configure a whitelist for code coverage analysis
105 * Renamed the `beStrictAboutTestSize` configuration option to `enforceTimeLimit`
106 * Printer-related CLI options now override printer-related configuration settings
107 
108 ### Removed
109 
110 * The `assertSelectCount()`, `assertSelectRegExp()`, `assertSelectEquals()`, `assertTag()`, `assertNotTag()` assertions have been removed
111 * The `--strict` commandline option and the XML configuration's `strict` attribute have been removed
112 * The code coverage blacklist functionality has been removed
113 * The PHPUnit_Selenium component is no longer bundled in the PHAR distribution
114 * The PHPUnit_Selenium component can no longer be configured using the `<selenium/browser>` element of PHPUnit's configuration file
115 * PHPUnit is no longer supported on PHP 5.3, PHP 5.4, and PHP 5.5
